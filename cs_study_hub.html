Modules 1-6: Basics & Control Logic

Module 1

Algorithms & Flowcharts

Overview: The foundational building blocks of programming. Before writing code, you need a logical plan.

  • Algorithm Definition: A step-by-step procedure to solve a problem. Important characteristics: Finiteness (must eventually stop), Definiteness (each step must be clear and unambiguous), Input (zero or more inputs), Output (at least one output), and Effectiveness (basic enough to be done exactly).
  • Flowcharts: A graphical representation of an algorithm. It helps in mapping out the logic visually before converting it into programming code.
  • Key Flowchart Symbols: Oval (Terminal/Start/Stop), Parallelogram (Input/Output), Rectangle (Process/Computation), Diamond (Decision/Condition Checking), Arrows (Flow of control).
  • Control Structures: Sequence (executing instructions one after the other), Selection (making a decision, e.g., if-else), and Iteration (repeating a set of instructions, e.g., loops).

Module 2

Introduction to C

Overview: C is a procedural programming language consisting of basic building blocks called tokens.

  • Structure of a C Program: Starts with Preprocessor Directives (like #include <stdio.h> to include standard input/output libraries), followed by Global Declarations, the main() function (where execution begins), and then user-defined functions.
  • C Tokens: The smallest individual units in a C program. They include Keywords (reserved words like int, if), Identifiers (names given to variables/functions), Constants (fixed values), Strings, Operators, and Special Symbols.
  • Rules for Identifiers: Must begin with a letter or an underscore (_). Cannot start with a digit. Cannot be a keyword. White spaces are not allowed.
  • Constants: Integer (whole numbers), Real/Floating-point (fractional numbers), Character (single character in single quotes 'A'), and String Constants (enclosed in double quotes "Hello").

Module 3

Operators & Expressions

Overview: Operators are symbols that tell the compiler to perform specific mathematical or logical manipulations.

  • Arithmetic Operators: +, -, *, /, and % (Modulo, which returns the remainder of an integer division). Note: Modulo cannot be used with floating-point numbers.
  • Relational & Logical: Relational (==, !=, >, <, etc.) compare values. Logical (&& AND, || OR, ! NOT) combine multiple conditions.
  • Bitwise Operators: Operate at the bit level. Includes AND (&), OR (|), XOR (^), Left Shift (<<), and Right Shift (>>). Useful for low-level programming.
  • Ternary (Conditional) Operator: A shorthand for if-else. Syntax: condition ? expression_if_true : expression_if_false.
  • Operator Precedence: Determines the order in which operators are evaluated. Unary operators usually have the highest precedence, followed by Arithmetic, Relational, Logical, and Assignment.

Module 4

Data Types & I/O

Overview: How memory is allocated for variables and how a program interacts with user inputs and screen outputs.

  • Primary Data Types: int (integers, typically 2 or 4 bytes), float (single-precision floating point, 4 bytes), double (double-precision floating point, 8 bytes), and char (single character, 1 byte).
  • Format Specifiers: Used in formatted I/O to indicate the data type. %d or %i for int, %f for float, %lf for double, %c for char, and %s for string.
  • Formatted vs Unformatted I/O: printf() and scanf() are formatted. Unformatted include getchar() and putchar() (for single characters), and gets() and puts() (for strings).
  • Type Casting: Converting a variable from one data type to another. Implicit conversion is done automatically by the compiler. Explicit conversion is done by the programmer using a cast operator, e.g., (float) 5 / 2.

Module 5

Control Statements

Overview: Mechanisms that allow a program to deviate from sequential execution based on conditions.

  • Branching Statements:
    • if / if-else: Basic two-way decision making.
    • Nested if-else / else-if ladder: Multiple condition checking.
    • switch: Multi-way branching based on the value of an integer or character expression. Cases must be unique.
  • Looping Statements:
    • Entry-Controlled Loops: Condition is checked before entering the loop (for loop, while loop). Use for when number of iterations is known.
    • Exit-Controlled Loop: Condition is checked at the end. The loop body executes at least once (do-while loop).
  • Jump Statements: break (exits the innermost loop or switch block immediately), continue (skips the rest of the current iteration and jumps to the next iteration in a loop).

Module 6

Arrays & Strings

Overview: Data structures used to store multiple values of the same type in contiguous memory locations.

  • Arrays: A fixed-size sequential collection of elements of the same data type. Elements are accessed using a 0-based index. For example, in int arr[5], the first element is arr[0] and the last is arr[4].
  • Multidimensional Arrays: Arrays of arrays. A 2D array is structured like a matrix with rows and columns. Accessed via two indices, e.g., matrix[row][col].
  • Strings: In C, strings are simply one-dimensional arrays of characters terminated by a special null character '\0'. This null character marks the end of the string.
  • String Manipulation Functions: Found in <string.h>.
    • strlen(): Returns length of string (excluding null).
    • strcpy(): Copies one string to another.
    • strcat(): Concatenates (appends) strings.
    • strcmp(): Compares two strings character by character.

Modules 7-12: Advanced C Concepts

Module 7

Pointers

Overview: Pointers are powerful tools that interact directly with computer memory.

  • Definition & Operators: A pointer is a variable that stores the memory address of another variable. The & (Address-of) operator returns the memory address of its operand. The * (Dereference) operator gives the value stored at that specific address.
  • Pointer Arithmetic: Incrementing a pointer (ptr++) doesn't just add 1 to the address. It adds the sizeof(dataType) to the address, making it point to the next memory location of that type. Example: if an int pointer holds address 1000, ptr++ makes it 1004 (assuming an int is 4 bytes).
  • Null Pointers / Void Pointers: A NULL pointer points to memory block 0 (nowhere), preventing accidental memory corruption. A void pointer is generic and has no associated data type, allowing it to hold references to any data type (must be typecast before dereferencing).

Module 8

Structures & Unions

Overview: User-defined data types that allow grouping variables of different data types under a single name.

  • Structure (struct): A composite data type. Every member of a structure has its own dedicated memory allocation. The total size of a structure is at least the sum of the sizes of its members. Members are accessed using the dot (.) operator, or the arrow (->) operator if using a pointer to the structure.
  • Union: Similar to a structure, but all members share the same memory location. The total size of a union is equal to the size of its largest member. You can only use one member at a time; updating one member overwrites the others. Useful for memory optimization.
  • Array of Structures: Extremely useful for maintaining databases. For instance, struct Student classRecord[60] creates an array capable of storing the distinct structures (records) of 60 students.

Module 9

Functions & Recursion

Overview: Breaking down a complex problem into smaller, manageable, and reusable sub-blocks of code.

  • Function Architecture: Consists of the Prototype (declaration advising compiler of return type and arguments), the Definition (the actual body/logic of the function), and the Call (where the function is practically used).
  • Parameter Passing:
    • Call by Value: A copy of the argument is passed. Changes made inside the function do NOT affect the original variable.
    • Call by Reference: The memory address (pointer) of the argument is passed. Changes made inside the function DO modify the original variable.
  • Recursion: A powerful technique where a function calls itself. It must have a Base Case (to stop the recursive looping) and a Recursive Case (breaking down the problem). Common examples include Factorial calculation and Fibonacci series generation.

Module 10

Storage Classes

Overview: Determine the scope, visibility, and lifetime/persistence of variables in C.

  • auto: The default behavior for all local variables declared inside a block or function. Created when block is entered, destroyed upon exit. Default value is garbage.
  • static: Variables that retain their value between multiple function calls. They are initialized only once. Default value is 0. Used for state preservation (like counting how many times a function is called).
  • extern: Used for global variables that are defined in another completely different C file, allowing cross-file variable sharing.
  • register: Hints to the compiler to store the variable in the CPU's register instead of RAM for extremely fast access. Usually reserved for frequently accessed variables like loop counters. Modern compilers often optimize this automatically.

Module 11

Dynamic Memory Allocation

Overview: Functions available in <stdlib.h> that allow allocating and freeing memory space at runtime (on the Heap).

  • malloc(): "Memory Allocation". Allocates a single large block of contiguous memory specified by the number of bytes. Leaves the memory uninitialized (garbage values). Returns a void pointer that requires type casting.
  • calloc(): "Contiguous Allocation". Takes two arguments: number of elements and size of each element. Crucially, it initializes all allocated memory spaces to zero.
  • realloc(): "Re-allocation". Used to resize a previously allocated memory block during runtime without losing the old data.
  • free(): Deallocates/releases dynamically allocated memory back to the operating system. Failing to use free() leads to "Memory Leaks", crashing the program over time.

Module 12

File Management

Overview: Storing data permanently in secondary storage devices because RAM is volatile.

  • File Modes: Passed to fopen().
    • "r": Read mode (errors if file doesn't exist).
    • "w": Write mode (creates a new file or overwrites an existing file entirely).
    • "a": Append mode (adds new data to the end of an existing file).
  • Core Operations: fopen() opens a file and returns a FILE*. fclose() safely closes it. I/O is handled via fprintf() / fscanf() for formatted text, or fread() / fwrite() for binary blocks of data.
  • Random Access: Navigating arbitrarily through a file. fseek() moves the file pointer to a specific byte location. ftell() reveals the current byte position. rewind() resets the pointer back to the very beginning of the file.
